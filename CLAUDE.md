## Project Overview

工时记录 (Daily Report) - A Qt desktop application for tracking work hours.

## Tech Stack

- **Desktop Framework**: Qt 6.6 Widgets (C++ backend)
- **Build System**: CMake 3.26+
- **Language**: C++20
- **Data**: QJsonDocument/QJsonObject for persistence, UUID for session IDs

## Project Structure

```
/root/DailyReport/
├── src/                 # C++ source code
│   ├── main.cpp         # Application entry point
│   ├── mainwindow.h/cpp # Main window class with UI logic
│   ├── datamodel.h/cpp  # Data models (WorkSession, DailyStatistics)

```

## Architecture

- **Single-process desktop application**: All logic runs in the main process
- **Data persistence**: Sessions stored as JSON files (organized by month in user data directory)
- **Qt Widgets**: Native UI components for cross-platform desktop experience

## Common Commands

```bash
# Configure build
mkdir -p build && cd build
cmake ..

# Build application
cmake --build .

# Run application
./DailyReport

# Linux deployment
cd /root/DailyReport
./package/linux/deploy.sh
```

## API/Features

- `startShift()` - Start a new work session
- `endShift(activity, workType)` - End current session
- `getSessions(date)` - Load sessions for a date
- `updateSession(date, sessionId, activity, workType)` - Edit session
- `deleteSession(date, sessionId)` - Delete session
- `exportData(date, format)` - Export to JSON/CSV
- `getTodayStatistics()` - Get today's total hours and session count
- `closeEvent()` - Auto-save active session on app close
